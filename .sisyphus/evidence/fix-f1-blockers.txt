## Bug 3 — OffsetCapper.js + OffsetEngine.js: Degenerate caps (start=end)
Files:
    - src/operations/OffsetCapper.js — added capBothSides() function
    - src/operations/OffsetContourBuilder.js — added skipCap option
    - src/operations/OffsetEngine.js — imports capBothSides, uses two-sided capping for open contours

APPROACH (Option B):
    - OffsetEngine._processSegmentsSync detects open source contours
    - Computes positiveSegments = buildOffsetContour(+d, skipCap=true)
    - Computes negativeSegments = buildOffsetContour(-d, skipCap=true)
    - Calls capBothSides(positiveSegments, negativeSegments, distance, capType)
    - capBothSides creates non-degenerate caps:
        * Flat cap: line from posStart → negStart (at original curve start)
        * Round cap: arc centered at midpoint of offset endpoints with proper radius
    - Full contour: posSegs → endCap → negSegsReversed → startCap

BEFORE (degenerate):
    startCap = capFlat(firstSeg.start, firstSeg.start);  // start=end → zero-length
    endCap = capFlat(lastSeg.end, lastSeg.end);            // start=end → zero-length

AFTER (correct):
    startCap = capFlat(negFirst.start, posFirst.start);   // connects two offset sides
    endCap = capFlat(posLast.end, negLast.end);            // connects two offset sides
