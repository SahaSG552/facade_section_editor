# LLM Wiki System - Schema

## Overview
This project uses the LLM Wiki pattern for knowledge management, inspired by [karpathy/llm-wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f).

## Architecture

### Three Layers

1. **Raw Sources** (`raw/`) - Immutable source documents
2. **Wiki** (`.wiki/`) - Generated knowledge pages
3. **Schema** (`WIKI.md`) - This file describes conventions

## Operations

### Ingest: Adding New Sources

1. Drop source file in `raw/[category]/`
2. Run: `skill({ name: "ingest-wiki" })`
3. LLM processes:
   - Creates summary page in `.wiki/sources/`
   - Updates `.wiki/index.md`
   - Adds cross-references to entity/concept pages
   - Logs action in `.wiki/log.md`

**Example:**
```bash
# Add Paper.js documentation
cp /path/to/paperjs docs.pdf raw/paperjs-docs/
skill({ name: "ingest-wiki" })
```

### Query: Finding Information

1. Ask question
2. LLM reads `.wiki/index.md` first
3. Searches relevant pages
4. Synthesizes answer with citations
5. Optionally files answer back as new wiki page

**Example:**
> "How do I implement V-Carve offset?"

→ LLM reads index → finds v-carve.md, operations.md → synthesizes answer

### Maintain: Health Check

1. Run: `skill({ name: "maintain-wiki" })`
2. LLM checks:
   - Contradictions between pages
   - Stale information superseded by new sources
   - Orphan pages (no inbound links)
   - Missing cross-references
3. Fixes issues found
4. Logs action in `.wiki/log.md`

## Page Conventions

### Naming
- Lowercase with hyphens: `v-carve.md`, `coordinate-systems.md`
- Use category folders: `entities/`, `concepts/`, `sources/`

### Frontmatter (optional)
```yaml
---
tags: [concept, offset]
date: 2026-04-15
sources: 2
last-updated: 2026-04-15
---
```

### Cross-References
- Use relative links: `[bits](entities/bits.md)`
- Prefix: supports, contradicts, related_to

### Content Structure
```markdown
# Page Title

## Overview
Brief description.

## Details
Main content...

## Related
- [bits.md](entities/bits.md) - Related entity
- [offset.md](concepts/offset.md) - Related concept

## Sources
- Source name or file reference
```

## Log Format

### Template
```markdown
## [[date]] action | Description
- Action performed

## [[date]] query | Search query
- Query: "what was searched"
- Found: relevant pages

## [[date]] lint | Health check
- Fixed: issues found
- Updated: pages modified

## [[date]] ingest | Source name
- Source: source description
- Extracted: key information
- Updated: affected pages
- Cross-ref: links created
```

### Parsing
```bash
# Last 5 entries
grep "^## \[" .wiki/log.md | tail -5
```

## MCP Search (qmd)

For full-text search across wiki:

```bash
# Install qmd
npm install -g qmd

# Start search server
qmd serve --path .wiki

# Search via API
curl http://localhost:8080/search?q=v-carve
```

## Obsidian Integration

### Setup
1. Open `.wiki/` as Obsidian vault
2. Enable Graph View plugin
3. Enable Dataview plugin

### Graph View
- Shows connections between pages
- Identifies hub pages and orphans
- Visualizes knowledge structure

### Dataview Queries
```dataview
LIST
FROM "entities"
WHERE tags
SORT file.name
```

## For Agents

All agents should understand this system:

### Workflow
1. **Check index first** - Start with `.wiki/index.md`
2. **Search wiki** - Read relevant pages before answering
3. **Ingest new sources** - Update wiki when learning new information
4. **Maintain** - Run lint periodically

### Key Files
- `.wiki/index.md` - Always read first
- `.wiki/log.md` - Recent activity
- This file - Schema reference

## Why This Works

1. **Knowledge compounds** - Each ingest makes the wiki richer
2. **Cross-references are built-in** - No need to re-discover connections
3. **Agents use it** - Consistent knowledge base for all agents
4. **Searchable** - qmd provides fast full-text search
